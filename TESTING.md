# 同步功能测试指南

## 测试环境准备

### 1. LeanCloud 测试应用
1. 登录 LeanCloud 控制台 (cn.leancloud.cn)
2. 创建测试应用，选择「华东」或「华北」节点
3. 获取 App ID 和 App Key
4. 开启「数据存储」服务
5. 开启「用户系统」服务

### 2. Android 测试环境
1. 修改 `LeanCloudManager.kt` 中的配置：
   ```kotlin
   private const val APP_ID = "你的测试App ID"
   private const val APP_KEY = "你的测试App Key"
   private const val SERVER_URL = "https://你的服务器地址.lc-cn-n1-shared.com"
   ```

2. 构建并安装应用到测试设备

### 3. 网页端测试环境
1. 修改 `web/src/utils/leancloud.js` 中的配置：
   ```javascript
   const APP_ID = '你的测试App ID'
   const APP_KEY = '你的测试App Key'
   const SERVER_URL = 'https://你的服务器地址.lc-cn-n1-shared.com'
   ```

2. 启动开发服务器：
   ```bash
   cd web
   npm install
   npm run dev
   ```

## 测试用例

### 测试用例 1：用户注册与登录

#### Android 端测试
1. 启动应用
2. 点击「登录」按钮
3. 切换到「注册」标签页
4. 输入用户名、邮箱、密码
5. 点击「注册」按钮
6. 验证：应显示注册成功，自动登录并跳转到主界面

#### 网页端测试
1. 访问 `http://localhost:3000`
2. 点击「登录」按钮
3. 切换到「注册」标签页
4. 输入用户名、邮箱、密码
5. 点击「注册」按钮
6. 验证：应显示注册成功，自动登录并跳转到任务列表

#### 跨端验证
1. 在 Android 端注册的用户
2. 在网页端使用相同邮箱密码登录
3. 验证：应能成功登录，显示相同用户名

### 测试用例 2：任务创建与同步

#### Android 端创建任务
1. 登录 Android 应用
2. 创建新任务：
   - 标题：测试任务 Android
   - 备注：这是从 Android 创建的任务
   - 优先级：高
   - 截止日期：明天
3. 保存任务
4. 验证：任务应显示在本地列表中

#### 网页端查看任务
1. 登录网页端（使用相同账号）
2. 刷新页面或等待自动同步
3. 验证：应能看到「测试任务 Android」
4. 验证：任务详情应与 Android 端一致

#### 网页端修改任务
1. 在网页端编辑「测试任务 Android」
2. 修改标题为「测试任务 Android（已修改）」
3. 标记为已完成
4. 保存修改

#### Android 端查看修改
1. 返回 Android 应用
2. 下拉刷新或等待自动同步
3. 验证：任务标题应已更新
4. 验证：任务状态应显示为已完成

### 测试用例 3：离线操作与同步

#### Android 离线创建
1. 关闭设备网络（WiFi/移动数据）
2. 在 Android 应用创建新任务：
   - 标题：离线创建的任务
   - 备注：这是在离线状态下创建的任务
3. 保存任务
4. 验证：任务应显示在本地列表中，同步状态为「待同步」

#### 恢复网络并同步
1. 重新开启网络
2. 等待自动同步或手动触发同步
3. 验证：任务同步状态应变为「已同步」

#### 网页端验证
1. 在网页端刷新页面
2. 验证：应能看到「离线创建的任务」

### 测试用例 4：冲突处理

#### 双端同时修改
1. 在 Android 端和网页端同时打开同一个任务
2. Android 端修改：更改标题为「Android 修改」
3. 网页端修改：更改标题为「网页端修改」
4. 两端同时保存

#### 冲突解决验证
1. 等待同步完成
2. 验证：根据冲突解决策略（当前为远程优先）
   - 如果网页端后保存，则显示「网页端修改」
   - 如果 Android 端后保存，则显示「Android 修改」
3. 检查同步状态：冲突应已解决

### 测试用例 5：删除与恢复

#### 网页端删除任务
1. 在网页端选择任意任务
2. 点击删除按钮
3. 确认删除
4. 验证：任务应从列表中消失

#### Android 端验证删除
1. 等待同步或手动刷新
2. 验证：任务不应再显示在 Android 端

#### LeanCloud 控制台验证
1. 登录 LeanCloud 控制台
2. 进入数据存储 > Task 表
3. 验证：删除的任务 `deleted` 字段应为 `true`

### 测试用例 6：性能测试

#### 大量数据同步
1. 在 Android 端批量创建 100 个测试任务
2. 触发同步
3. 记录同步完成时间
4. 验证：所有任务应成功同步到云端

#### 网页端加载性能
1. 在网页端打开任务列表
2. 记录页面加载时间
3. 验证：100 个任务应在 3 秒内加载完成

#### 内存使用
1. 监控 Android 应用内存使用
2. 同步过程中内存不应显著增加
3. 同步完成后内存应恢复正常

2. 测试：
   - 应用崩溃防护
   - 错误日志记录
   - 数据恢复机制

## 问题排查指南

### 常见问题

#### Q: 同步失败，显示网络错误
A: 
1. 检查网络连接
2. 验证 LeanCloud 配置是否正确
3. 检查服务器地址是否可访问

#### Q: 登录成功但看不到任务
A:
1. 检查用户 ID 是否一致
2. 验证 Task 表的 userId 字段
3. 检查数据权限设置

#### Q: 同步状态一直显示「待同步」
A:
1. 检查网络连接
2. 查看错误日志
3. 尝试手动触发同步

#### Q: 双端数据不一致
A:
1. 检查冲突解决策略
2. 查看 updatedAt 时间戳
3. 检查时区设置
